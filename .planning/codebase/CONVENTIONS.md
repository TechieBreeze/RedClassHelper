# Coding Conventions

## Dart/Flutter Style

### Widget Architecture

- **All screens are `StatelessWidget`** — state lives in Riverpod providers, not widget state
- **Private helper widgets** use `_` prefix (`_SectionHeader`, `_ModeTile`) — keeps build tree readable
- **No `StatefulWidget`** anywhere in current codebase
- **`const` constructors** everywhere possible (enforced by `prefer_const_constructors` lint)

### Layout Pattern (from HomeScreen)

```dart
Scaffold(
  appBar: AppBar(title: Text('...')),
  body: LayoutBuilder(                    // responsive width
    builder: (context, constraints) {
      // 3 breakpoints: <600 (compact), <840 (medium), ≥840 (expanded)
      // Expanded: maxWidth = 720, centered via Center + ConstrainedBox
      return Center(
        child: ConstrainedBox(
          constraints: BoxConstraints(maxWidth: ...),
          child: SingleChildScrollView(   // scrollable body
            padding: ...,
            child: Column(
              crossAxisAlignment: CrossAxisAlignment.stretch,
              children: [ ... ],
            ),
          ),
        ),
      );
    },
  ),
);
```

### Card + InkWell Pattern

All tappable cards follow:
```dart
Card(
  child: InkWell(
    onTap: () => context.go('/route'),
    borderRadius: BorderRadius.circular(12),
    child: Padding(
      padding: const EdgeInsets.all(16),
      child: Row(children: [...]),   // or Column
    ),
  ),
)
```

### Trailing Commas

`require_trailing_commas` lint is enforced — all multi-line parameter lists, lists, and maps end with trailing commas to produce clean diffs.

## State Management

### Provider Patterns

```dart
// Keep-alive singleton (DB, PathResolver):
@Riverpod(keepAlive: true)
Future<AppDatabase> appDatabase(Ref ref) async { ... }

// Non-keep-alive (auto-disposed when no listeners):
@riverpod
Future<List<QuestionBank>> questionBanks(Ref ref) async { ... }
```

### Provider Override in main()

```dart
ProviderScope(
  overrides: [
    pathResolverProvider.overrideWith((ref) async => resolver),  // pre-resolved instance
  ],
  child: const RedClassApp(),
)
```

## Database

### Table Definition

```dart
@DataClassName('Question')      // generates Question data class (not Questions)
class Questions extends Table {
  TextColumn get id => text()();
  TextColumn get bankId => text()
      .named('bank_id')
      .references(QuestionBanks, #id, onDelete: KeyAction.cascade)();

  @override
  Set<Column> get primaryKey => {id};
}
```

- All primary keys are `TextColumn` (UUID strings, not auto-increment integers)
- Foreign keys use `KeyAction.cascade` for cleanup on parent deletion
- JSON columns store serialized lists as `text()` with snake_case column names

### Query Access

```dart
// All queries go through db.* getters auto-generated by drift:
await db.select(db.questionBanks).get();
await db.into(db.questionBanks).insert(QuestionBanksCompanion.insert(...));
(db.select(db.questionBanks)..where((t) => t.id.equals(id))).getSingle();
```

## File Organization

- Feature screens live at `lib/features/{feature}/presentation/{feature}_screen.dart`
- Each feature is a self-contained directory — no cross-feature imports
- Navigation between features uses GoRouter paths, not direct widget imports
- Generated files (`*.g.dart`, `*.freezed.dart`) are gitignored but committed (needed for CI without codegen)

## Lint Rules

| Rule | Rationale |
|------|-----------|
| `prefer_const_constructors` | Reduce rebuilds, enable compile-time optimization |
| `prefer_final_locals` | Immutability by default |
| `avoid_print` | Use `debugPrint` or proper logging |
| `require_trailing_commas` | Clean diffs, auto-format friendliness |
| `sort_child_properties_last` | Consistent widget ordering |
| `invalid_annotation_target: ignore` | Required for freezed + json_serializable annotations |

## Import Order

No explicit convention enforced yet. Current practice: Flutter SDK → packages → relative imports.
