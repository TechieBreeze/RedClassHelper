<objective>
Add a JSON import fast-track method to ImportNotifier. When the user picks a `.json` file via
the import screen, the import pipeline reads the file, validates it against the D-01 format,
converts to DB entities using the export service's userJsonToEntities(), and commits directly
to the database -- skipping the editing/preview phase entirely (D-05). Duplicate bank names
trigger silent replacement inside a DB transaction (D-06).

Purpose: JSON import is the desktop-to-desktop transfer protocol. This makes it a single-click
operation with no manual editing needed, since JSON format data is already precise.

Output: Modified import_notifier.dart + comprehensive import unit tests.

    ### Method 1: `Future<void> importJsonFile()` (new)

    ```
    Future<void> importJsonFile() async {
      if (state.files.isEmpty) return;

      final filePath = state.files.first.path;

      // Guard 1: File size check (reject >10MB before reading into memory)
      final file = File(filePath);
      final fileSize = await file.length();
      if (fileSize > 10 * 1024 * 1024) {
        state = state.copyWith(
          phase: ImportPhase.idle,
          error: 'JSON 文件过大（超过 10MB），请检查文件内容',
        );
        return;
      }

      // Guard 2: Read and parse JSON
      state = state.copyWith(phase: ImportPhase.parsing, progress: 0.1);
      final jsonString = await file.readAsString();

      Map<String, dynamic> jsonData;
      try {
        jsonData = jsonDecode(jsonString) as Map<String, dynamic>;
      } on FormatException catch (e) {
        state = state.copyWith(
          phase: ImportPhase.idle,
          error: 'JSON 格式错误：${e.message}',
        );
        return;
      }

      // Guard: provider may have been disposed during file I/O
      if (!_mounted) return;

      // Step 3: Validate top-level structure (before DB operations)
      if (jsonData['name'] is! String || (jsonData['name'] as String).isEmpty) {
        state = state.copyWith(
          phase: ImportPhase.idle,
          error: 'JSON 文件缺少有效的题库名称（name 字段）',
        );
        return;
      }
      if (jsonData['version'] is! String) {
        state = state.copyWith(
          phase: ImportPhase.idle,
          error: 'JSON 文件缺少版本号（version 字段）',
        );
        return;
      }
      if (jsonData['questions'] is! Map || (jsonData['questions'] as Map).isEmpty) {
        state = state.copyWith(
          phase: ImportPhase.idle,
          error: 'JSON 文件不包含题目数据（questions 字段为空或缺失）',
        );
        return;
      }

      state = state.copyWith(progress: 0.3);

      // Step 4: Convert and commit in a DB transaction (D-06 atomic replacement)
      final db = await ref.read(appDatabaseProvider.future);
      if (!_mounted) return;

      try {
        final bankId = _uuid.v4();
        final now = DateTime.now();
        final jsonBankName = jsonData['name'] as String;

        await db.transaction(() async {
          // 4a. Detect and replace duplicate bank (D-06: silent, no dialog)
          final existingBank = await (db.select(db.questionBanks)
            ..where((b) => b.name.equals(jsonBankName))
          ).getSingleOrNull();

          final int oldCount;
          if (existingBank != null) {
            oldCount = existingBank.questionCount;
            // Cascade delete: removing bank deletes all its questions + ledger + attempts
            await (db.delete(db.questionBanks)
              ..where((b) => b.id.equals(existingBank.id))
            ).go();
          } else {
            oldCount = 0;
          }

          state = state.copyWith(progress: 0.5);

          // 4b. Convert JSON to entities (uses Plan 01's userJsonToEntities)
          final converted = userJsonToEntities(jsonData, bankId);

          // 4c. Insert QuestionBank
          await db.into(db.questionBanks).insert(
            QuestionBanksCompanion.insert(
              id: bankId,
              name: converted.bankName,
              source: filePath,
              questionCount: converted.questions.length,
              createdAt: now,
              updatedAt: now,
            ),
          );

          state = state.copyWith(progress: 0.7);

          // 4d. Create ParseJob for summary screen
          final job = ParseJobsCompanion.insert(
            id: _uuid.v4(),
            sourcePath: filePath,
            status: 'succeeded',
            progress: 1.0,
            resultCount: converted.questions.length,
            createdAt: now,
            updatedAt: now,
          );
          await db.into(db.parseJobs).insert(job);

          // 4e. Insert questions one by one with progress
          for (var i = 0; i < converted.questions.length; i++) {
            await db.into(db.questions).insert(converted.questions[i]);
            state = state.copyWith(
              progress: 0.7 + 0.3 * ((i + 1) / converted.questions.length),
            );
          }
        });

        // Step 5: Transition to done state
        state = state.copyWith(
          phase: ImportPhase.done,
          committedCount: converted.questions.length,
          progress: 1.0,
          bankId: bankId,
          bankName: converted.bankName,
        );
      } on FormatException catch (e) {
        // Validation errors from userJsonToEntities (bad key format, invalid answer_type, etc.)
        state = state.copyWith(
          phase: ImportPhase.idle,
          error: '题库数据验证失败：${e.message}',
        );
      } on Exception catch (e) {
        state = state.copyWith(
          phase: ImportPhase.idle,
          error: '导入失败：${e.toString()}',
        );
      }
    }
    ```

    **CRITICAL behaviors (D-05, D-06):**

    - Replacement is in a `db.transaction()` -- delete old + insert new atomically
    - No confirmation dialog shown to user
    - Error messages are in Chinese, user-friendly

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system → `importJsonFile()` | Untrusted .json file content crosses into domain logic |
| JSON key field → DB insert | Crafted `key` value could carry injection payload |
| Bank name → DB query | Bank name used in WHERE clause for duplicate detection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Denial of Service | `importJsonFile()` oversized file | mitigate | Check file size via `File.length()` before `readAsString()`; reject files >10,485,760 bytes (10MB) with clear Chinese error message |
| T-05-07 | Tampering | `importJsonFile()` key field injection | mitigate | Reject at `userJsonToEntities()` layer: regex `^[A-H]+$` on key field; any non-A-H character throws FormatException caught by importJsonFile |
| T-05-08 | Tampering | `importJsonFile()` malformed JSON | mitigate | `jsonDecode()` wrapped in try/catch for FormatException; top-level structure validation (name, version, questions) before any DB operation |
| T-05-09 | Repudiation | `importJsonFile()` duplicate bank replacement | accept | D-06 requires silent replacement (no confirmation dialog). Replacement is atomic inside `db.transaction()` -- if commit fails, old bank is preserved. SnackBar feedback shows "已替换已有题库「bankName」" for user awareness. |
| T-05-10 | Information Disclosure | Error messages from failed import | accept | Error messages describe validation failures (missing fields, bad format) but do not leak file paths or DB internals. Messages are in Chinese, user-facing. |
</threat_model>
