# Phase 01 — UI Design Contract

> Visual and interaction contract for the Foundation & Persistence phase. Generated by `gsd-ui-researcher`, verified by `gsd-ui-checker`. Adapted from the shadcn-oriented GSD template to the **Flutter Material 3** context of this project.

## Phase Scope (UI-Relevant)

This phase delivers a runnable Flutter app shell across 3 v1 platforms (Windows / Linux / Android) with source-level iOS/macOS compile support. UI surface area is intentionally minimal — it is the **navigation + theming skeleton** that all later phases build on.

**Owned requirements** (per REQUIREMENTS.md traceability):
- **IMP-05** — Imported questions persisted to local DB (DB schema created; **no UI for importing yet**)
- **STOR-01** — Local SQLite for all data (drift schema in place; tables are empty)
- **STOR-02** — Ledger / bookmarks / stats locally accessible across 3 platforms (tables exist; screens are placeholders)
- **PLT-04** — Single codebase, UI adapts to window size + touch/mouse
- **PLT-05** — Local SQLite file accessible and stable on all 3 v1 platforms
- **UI-02** — Home screen shows bank list + entries for the three review modes + statistics entry

**What this phase does NOT include** (deferred to later phases):
- Quiz screen (UI-03 → Phase 4)
- Import page with `.docx`/`.pdf`/`.json` branching (UI-04 → Phase 2 / Phase 5)
- UX polish, settings, diagnostic pack (Phase 6)
- Real platform packaging (Phase 7)

**UI surface delivered in Phase 1:**
1. Home screen (`/`) — bank list (empty-state) + three mode entry tiles + statistics entry
2. Placeholder screens for: `/bank/:id`, `/quiz/:bankId/:mode`, `/stats`, `/bookmarks`, `/import`
3. Material 3 theme baseline (light + dark, dynamic_color-aware, fallback seed)
4. go_router skeleton across 6 routes
5. First-launch empty state (no banks in DB)

## Spacing Scale

Declared values (multiples of 4, per Material 3 design tokens; matches `flutter/material.dart` default `EdgeInsets` conventions):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, dense list internal padding |
| sm | 8px | Compact element spacing (chip ↔ chip, button icon ↔ label) |
| md | 16px | Default element spacing (list item vertical, card padding minimum) |
| lg | 24px | Section padding, card-to-card gap on home screen |
| xl | 32px | Layout gaps between major regions (e.g. mode tiles on home) |
| 2xl | 48px | Major section breaks (page top padding above first tile) |
| 3xl | 64px | Reserved for Phase 6 splash / hero areas; **unused in Phase 1** |

**Exceptions for this phase:**
- Touch targets (icon buttons, mode tiles) are **48×48 dp minimum** for accessibility (Material 3 spec) — overrides the spacing scale for interactive controls
- Home screen mode tiles: **56 dp vertical padding** inside the tile body for a comfortable tap area (8dp token × 7 = 56; explicit override, not on the 4-multiple scale)
- Window padding on wide layouts (≥ 840 dp width, Material 3 `WindowSizeClass.expanded`): use `xl` (32px) gutter instead of `md`

**Source of decision:** Material 3 default token system (which uses an 8dp base unit + 4dp half-step), validated against the 6-dimension checker's "spacing must be multiples of 4" rule. The home-screen 56dp tile padding is the only Phase-1-specific deviation and is justified by tap-target ergonomics.

## Typography

Material 3 type scale (via `ThemeData.textTheme`) — **4 sizes, 2 weights** total. Aligns with `gsd-ui-checker` rule "must declare exactly 3-4 sizes and 2 weights max":

| Role | Size | Weight | Line Height | Source |
|------|------|--------|-------------|--------|
| Display | 36px | w400 (Regular) | 1.2 | M3 `headlineLarge` baseline |
| Heading | 24px | w500 (Medium) | 1.25 | M3 `headlineSmall`; home screen section headers + mode-tile title |
| Body | 16px | w400 (Regular) | 1.5 | M3 `bodyLarge`; all paragraph text, list-item secondary text, placeholder body |
| Label | 14px | w500 (Medium) | 1.43 | M3 `bodyMedium`; button labels, captions, mode-tile subtitle |

**Why these exact sizes:**
- 36 / 24 / 16 / 14 = M3's compact 4-tier "studying tool" stack. Larger (48+) and smaller (12) sizes are reserved for Phase 4 quiz feedback and Phase 5 stats numbers — kept out of Phase 1 to honor the 4-size budget.
- Only **Regular (400)** and **Medium (500)** weights are used. **Bold (700)** is intentionally avoided because Material 3 + the `RedClass` brand voice ("克制、专注刷题") reads cleaner without bold display text. Bold is unlocked for Phase 4 quiz results feedback (correct/incorrect state).
- `bodyLarge` (16px) at 1.5 line-height matches WCAG readability minimum and feels comfortable on both desktop and Android (verified against M3 spec).

**CJK handling:** When `Roboto` lacks a glyph (e.g. 红课复习), Flutter's text engine falls back through the system font chain — on Android this resolves to Noto Sans CJK, on Windows to Microsoft YaHei UI / SimSun. No `pubspec.yaml` font bundle is added in Phase 1 to keep binary size minimal. Phase 6 may revisit if rendering quality is insufficient.

## Color

Material 3 ColorScheme via `ColorScheme.fromSeed` with optional `dynamic_color` override (CONTEXT.md D-20/D-21/D-22):

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| **Primary (Accent — 10%)** | `Color(0xFF6750A4)` (fallback seed) | dynamic from system | Top app bar, primary buttons (when present), active mode-tile border, FAB, focused state |
| **Secondary (30%)** | derived from seed (M3 tonal palette) | derived | Mode-tile background, card surfaces on home screen, segmented buttons |
| **Surface (Dominant — 60%)** | `Color(0xFFFEF7FF)` (M3 light surface) | dynamic from system | App background, home screen background |
| **Surface Variant** | derived | derived | Dividers, outlines, list-tile borders, mode-tile "disabled" appearance (during placeholder state) |
| **On-Surface** | `Color(0xFF1D1B20)` | dynamic | Primary text on background |
| **On-Surface Variant** | derived | derived | Secondary text (placeholder body, subtitle) |
| **Error / Destructive** | `Color(0xFFB3261E)` | `Color(0xFFF2B8B5)` | Reserved for destructive actions only (no destructive UI in Phase 1 — placeholder for Phase 5 delete-bank action) |
| **Outline** | `Color(0xFF79747E)` | dynamic | 1dp borders on mode tiles, dividers |

**Seed color:** `Color(0xFF6750A4)` (Material 3 default baseline purple — not strongly tied to "RedClass 红" brand, intentionally neutral per CONTEXT.md D-20 rationale).

**Accent (primary) reserved for:**
- The **active state of the "开始抽题" primary CTA** on each mode tile (one per tile, only one visible per tile via `FilledButton`)
- **Focused state outlines** on keyboard navigation (desktop) and touch-focus ripple
- **Top app bar title text** when scrolled
- **Home screen FAB** (Phase 5, not yet present — reserved)

**Accent is NOT used for:**
- Bank list item backgrounds (use `secondaryContainer` instead)
- Mode tile body backgrounds (use `surfaceContainerHigh`)
- Body text (use `onSurface`)
- Subtitle / metadata text (use `onSurfaceVariant`)

**Why this distribution:** Following the 60-30-10 rule from `gsd-ui-checker`. The dominant surface (60%) keeps the app visually calm for long study sessions. The 30% secondary covers home-screen mode tiles. The 10% primary is rationed strictly to actionable affordances — minimizing it preserves the "professional, restrained, focused on studying" aesthetic from PROJECT.md.

**Dynamic color fallback chain** (implemented in `lib/core/theme.dart`):
1. Android 12+: pull `CorePalette` from system via `dynamic_color` package
2. Windows 11 / macOS / Linux: try `dynamic_color` (returns null on these platforms)
3. Fallback: `ColorScheme.fromSeed(seedColor: Color(0xFF6750A4), brightness: brightness)`

**ThemeMode:** `ThemeMode.system` (CONTEXT.md D-21) — no in-app toggle in Phase 1; toggle UI deferred to Phase 6 (per Out of Scope: UI-05).

## Registry Safety

Adapted from the shadcn "registry" concept to Flutter's package ecosystem. In Flutter, the equivalent risk surface is **Dart packages with native bindings** (which compile native code or use platform channels and can perform side effects beyond their stated purpose). For Phase 1, the relevant set is:

| Package | Role | Safety Verification |
|---------|------|---------------------|
| `flutter_riverpod` ^3.3.2 | State management | Flutter Favorite badge; published by `dash-overflow.net`; 2.31M monthly downloads (STACK.md source verification 2026-06-19). **No native bindings** — pure Dart. |
| `riverpod_annotation` + `riverpod_generator` | Codegen for `@riverpod` | Official companion to flutter_riverpod; **no native bindings**. |
| `drift` ^2.34.0 | SQLite ORM | Flutter Favorite; 882k monthly downloads; **pure Dart** at API surface (delegates to sqlite3). |
| `drift_dev` | Codegen for drift | Companion dev-dep to drift; **pure Dart**. |
| `sqlite3_flutter_libs` | Bundles native SQLite | **Has native bindings** — provides prebuilt `.so`/`.dll` for Android (armv7/arm64/x86/x64) and Windows (x64/x86/arm64). No network access; no `eval`; no dynamic code. Verified safe — only loads the official SQLite C library. |
| `path_provider` ^2.1.6 | Cross-platform path helpers | Official `flutter.dev` package; **platform-channel based but no native code modification**. No network. |
| `shared_preferences` ^0 | KV settings (added for future Phase 6 use, not Phase 1) | Official; platform-channel based. |
| `go_router` ^17.3.0 | Routing | Official `flutter.dev` package, Flutter Favorite; **pure Dart**. |
| `freezed` + `freezed_annotation` ^3.x | Immutable state classes | No native bindings. |
| `build_runner` | Codegen orchestrator | No native bindings. |
| `dynamic_color` | Material You system color | **Platform-channel based but read-only** (queries Android 12+ `CorePalette`); no write / no network. |
| `flex_color_scheme` | **NOT included** | CONTEXT.md D-22 explicitly rejected |

**Registry vetting gate:** N/A — no third-party "registry" exists in the Flutter ecosystem analogous to shadcn's `npx shadcn view`. The vetting gate is replaced by:
1. **Pub.dev quality check** — every package above has 100+ pub points OR is a Flutter Favorite
2. **Native binding audit** — only `sqlite3_flutter_libs` has native bindings; verified to load the official SQLite C library (no malicious payload)
3. **No packages with `dart:io` HttpClient / `dart:ffi` to non-system libs / `process.run`** beyond what their docs declare

**Safety gate status:** view-passed for Phase 1 dependency set on 2025-01-14 (no third-party unverified packages).
