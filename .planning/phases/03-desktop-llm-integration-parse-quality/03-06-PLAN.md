<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (URL TextField) → AddModelDialog | Untrusted URL input — validate HTTPS scheme and .gguf extension |
| User file picker → GGUF Validator | Untrusted local file — magic number validation before import |
| UI → Download Provider | UI triggers downloads via provider methods — gated on Platform check |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06-01 | Tampering | add_model_dialog.dart URL input | mitigate | Validate URL format: must be https://, must end with .gguf. Shows inline error for invalid input. Prevents user from downloading non-model files. |
| T-03-06-02 | Tampering | add_model_dialog.dart local file | mitigate | GgufValidator magic number check prevents non-GGUF files. File picker filters for .gguf extension on some platforms. |
| T-03-06-03 | Spoofing | model_card.dart installed badge | accept | "已安装" badge is based on file existence in modelsDir. A user could manually place a file there. The badge is informational, not a security gate. |
| T-03-06-04 | Denial of Service | model_card.dart concurrent downloads | mitigate | Single download enforced by ModelDownloadNotifier. UI reflects this: other download buttons show "等待中" disabled state. |
| T-03-06-05 | Information Disclosure | settings_screen.dart | accept | Settings page shows model names and file sizes. No PII or secrets. |
</threat_model>
