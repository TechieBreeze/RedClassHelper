# Phase 04 — UI Design Contract

> Visual and interaction contract for the quiz core and wrong-question ledger phase.
> Generated by gsd-ui-researcher from upstream CONTEXT.md decisions, existing codebase patterns, and Claude's discretion areas.

---

| Property | Value |
|----------|-------|
| Tool | material3 |
| Framework | Flutter 3.44+ (StatelessWidget + Riverpod) |
| Component library | Material 3 (`useMaterial3: true`) |

## Spacing Scale

Declared values (must be multiples of 4). All values in logical pixels.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-label gap, chip padding, badge offset |
| sm | 8px | Compact element spacing, option-card gap, keyboard-hint padding |
| md | 16px | Default element spacing, Card inner padding, ListTile gap |
| lg | 24px | Section vertical gaps, section header spacing, quiz feedback spacing |
| xl | 32px | Horizontal layout padding (expanded desktop breakpoint) |
| 2xl | 48px | CTA button minimum height (`Size.fromHeight(48)`), summary icon spacing |
| 3xl | 64px | Summary page success icon size |

**Exceptions:** 12px used between home screen mode tiles (sm-md bridge). QuizScreen option cards use 12px vertical gap between options for tighter visual grouping.

## Typography

Mapped to Material 3 `TextTheme` tokens. No custom font override — uses M3 defaults.

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Body | `bodyMedium` | 14px | 400 | 1.5 | Descriptions, subtitles, keyboard hint, chip label |
| BodyLarge | `bodyLarge` | 16px | 400 | 1.5 | Option text, bank-list metadata, summary detail values |
| Heading | `headlineSmall` | 24px | 500 (w600 in summary) | 1.2 | Section headers (`_SectionHeader`), quiz progress counter, summary title |
| Display | `headlineLarge` | 36px | 400 | 1.2 | Main page titles (when needed) |

**Quiz-specific type treatments:**
- Question stem: `bodyLarge` (16px) at weight 500 — for readability of Chinese academic text
- Option labels (A/B/C/D): `titleMedium` (16px) at weight 700 — bold prefix on option rows
- Progress text ("第 3/20 题"): `labelLarge` (14px) at weight 500 — compact but legible
- Keyboard shortcut hint: `labelSmall` (11px) at weight 400, semi-transparent (opacity 0.6)

Source: M3 defaults confirmed in `lib/core/theme.dart` comment (line 30-32). Quiz-specific treatments are Claude's discretion based on D-01/D-04/D-06.

## Color

Material 3 `ColorScheme` tokens driven by dynamic color (`harmonized()`) with `kSeedColor = Color(0xFF6750A4)` fallback. The scheme adapts to the user's OS accent color when available (Windows accent color, Linux GTK theme).

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `scheme.surface` | Scaffold background, AppBar background |
| Secondary (30%) | `scheme.surfaceContainerHighest` (via CardTheme) | Cards (elevation 1, 12px radius), option cards |
| Accent (10%) | `scheme.primary` | **Reserved for:** primary CTA buttons (FilledButton), LinearProgressIndicator active track, selected option card fill, "已加入错题本" chip background |
| Correct | `Colors.green.shade600` | Correct answer icon (✔), correct option card background tint |
| Wrong | `Colors.red.shade600` | Wrong answer icon (✘), wrong selected option border/background, error state text |
| Neutral | `Colors.orange.shade700` | System warning indicator (not used in quiz; exists from import module) |
| Surface tint | `scheme.surfaceTint` | AppBar scrolled-under elevation |

**Accent reserved for:** Primary CTA buttons only. Option selection states use `scheme.primaryContainer` (the lighter variant). The accent color must NOT be used for decorative elements, dividers, or non-interactive text.

**Quiz feedback color contract (D-04):**
- **Correct option** (any): green background tint (`Colors.green.withOpacity(0.15)`) + ✔ check icon at row end
- **Wrong user-selected option**: red border (`Colors.red.shade600`, 2px) + ✘ icon at row end
- **Unselected correct option** (when user picked wrong): same green background as correct, to show the right answer
- **Unselected options** (when answer revealed): remain default card surface color, slightly muted (opacity 0.5)
- **"已加入错题本" chip** (D-15): `scheme.errorContainer` background with `scheme.onErrorContainer` text

Source: CONTEXT.md D-04, existing theme.dart CardTheme (elevation 1, radius 12). Feedback tint values are Claude's discretion.

```
│                                      │
│  ──────────────────────────────────  │
│  快捷键: A B C D 选择 · 空格 提交    │ ← labelSmall, opacity 0.6
│  · → 下一题                          │    fixed at bottom, over surface
└──────────────────────────────────────┘
```

**Submit flow (instant mode, D-02 default):**
1. User clicks option or presses A/B/C/D key
2. Option shows `scheme.primaryContainer` selection highlight
3. Answer is immediately submitted
4. All options reveal correct/incorrect state (D-04 colors)
5. If wrong: "已加入错题本" chip animates in (slide + fade, 200ms), auto-dismisses after 1.5s
6. After 2s (D-03 default): auto-advance to next question with crossfade (300ms)

**Submit flow (confirm mode, D-02 toggle):**
1. User clicks option — shows selection highlight
2. User presses Space key or taps "确认提交" FilledButton below options
3. Feedback reveals as above
4. If auto-advance enabled: 2s delay then next question
5. If manual-advance: user presses → key or taps "下一题" button

**States:**
- **Loading:** Centered CircularProgressIndicator + "加载题目..."
- **Empty (bank has 0 questions):** Centered text "该题库暂无题目" + "返回" OutlinedButton
- **Quiz complete:** Navigate to QuizSummaryScreen (go_router)
- **Error (question fetch fails):** SnackBar + retry button

## Interaction Contracts

### Keyboard Shortcuts (D-06)

| Key | Action | Context |
|-----|--------|---------|
| A | Select option A | Quiz screen, before answer |
| B | Select option B | Quiz screen, before answer |
| C | Select option C | Quiz screen, before answer |
| D | Select option D | Quiz screen, before answer |
| Space | Submit answer (confirm mode) / NOP (instant mode) | Quiz screen, option selected |
| → (Right Arrow) | Next question (manual advance mode) | Quiz screen, answer revealed |
| Escape | Navigate back (if allowed by quiz state) | Any quiz sub-screen |

Keyboard shortcut hint bar:
- Fixed at bottom of quiz screen, above system chrome
- Background: transparent, text: semi-transparent `onSurface.withOpacity(0.6)`
- Font: `labelSmall` (11px)
- Text: "快捷键: A B C D 选择 · 空格 提交 · → 下一题"
- Show on desktop only (`Platform.isWindows || Platform.isLinux`)
- Do NOT cover option area — place below the option list or as an overlay bar

### "已加入错题本" Chip Animation (D-15)

- **Trigger:** User answers incorrectly
- **Appearance:** Slide up from below the option list + fade in, duration 200ms, ease-out
- **Position:** Below the option list, above keyboard hint
- **Visibility:** Auto-dismiss after 1.5s with fade out (200ms)
- **Style:** `Chip` widget, `scheme.errorContainer` background, `scheme.onErrorContainer` label
- **Label:** "已加入错题本"
- **Icon:** `Icons.bookmark_added` (16px) on leading

### Wrong Count Badge (D-14)

- **Position:** Top-right corner of `_ModeTile` card, overlapping the card edge
- **Style:** `Badge` widget (Material 3) or `Container` with circular shape, `scheme.error` background, white text
- **Content:** Number (active wrong count from `LedgerRepository.getActiveCount()`)
- **Visibility:** Hidden when count is 0
- **Update:** Reactively via `wrongQuestionsProvider` StreamProvider

### Option Card Interaction States

| State | Background | Border | Icon | Cursor |
|-------|------------|--------|------|--------|
| Default | `scheme.surfaceContainerHighest` | none | Letter prefix only | pointer |
| Hover (desktop) | `scheme.surfaceContainerHighest` + 4% overlay | none | Letter prefix only | pointer |
| Selected (pre-submit) | `scheme.primaryContainer` | none | Letter prefix only | default |
| Correct (post-submit) | `Colors.green.withOpacity(0.15)` | none | ✔ `Icons.check_circle` (green.shade600) | default |
| Wrong selected (post-submit) | `scheme.surfaceContainerHighest` | 2px `Colors.red.shade600` | ✘ `Icons.cancel` (red.shade600) | default |

| Dimmed-unselected (post-submit) | `scheme.surfaceContainerHighest` opacity 0.5 | none | none | default |
