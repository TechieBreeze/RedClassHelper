       **parse() method logic (D-09: 3 retries then fail):**
       ```dart
       @override
       Future<ParseCandidate> parse(String rawText, {String? bankName}) async {
         String lastError = '';
         for (int attempt = 1; attempt <= maxRetries; attempt++) {
           try {
             return await _attemptParse(rawText, bankName: bankName);
           } on LlmTimeoutException {
             lastError = 'timeout on attempt $attempt';
             if (attempt == maxRetries) {
               throw LlmRetryExhaustedException(attempts: maxRetries, lastError: lastError);
             }
           } on LlmJsonParseException {
             // JSON parse failure is NOT retried — the LLM gave us non-JSON despite GBNF
             rethrow;
           } on LlmConnectionException {
             // Connection failure IS retried — server may be restarting
             lastError = 'connection refused on attempt $attempt';
             if (attempt == maxRetries) {
               throw LlmRetryExhaustedException(attempts: maxRetries, lastError: lastError);
             }
           }
         }
         throw LlmRetryExhaustedException(attempts: maxRetries, lastError: lastError);
       }
       ```

       **Error mapping:**
       - `TimeoutException` (dart:async) → `LlmTimeoutException(timeout: timeout, serverUrl: serverUrl)`
       - `SocketException` / `HttpException` / `ClientException` → `LlmConnectionException(serverUrl: serverUrl, originalError: e.message)`
       - `FormatException` from jsonDecode → `LlmJsonParseException(rawResponse: body, parseError: e.message)`
       - Non-200 status code → included in retry count, treats as transient failure

       ```dart
       Map<String, dynamic> _buildJsonSchema() {
         return const {
           'type': 'object',
           'properties': {
             'title': {'type': 'string', 'minLength': 1},
             'type': {'type': 'string', 'enum': ['single', 'multiple', 'truefalse', 'unknown']},
             'options': {
               'type': 'array',
               'items': {'type': 'string'},
               'minItems': 2,
               'maxItems': 8,
             },
             'answer': {'type': 'string', 'minLength': 1},
             'explanation': {'type': 'string'},
           },
           'required': ['title', 'type', 'options', 'answer'],
           'additionalProperties': false,
         };
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → llama.cpp server | localhost:8080 — local-only, no authentication. Input (rawText) is untrusted (could contain prompt injection via crafted .docx). LLM output (response.content) is untrusted (could be non-JSON). |
| LLM output → ParseCandidate | jsonDecode boundary — format validation before constructing ParseCandidate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Spoofing | llama.cpp endpoint | accept | Server runs on localhost only, not reachable from network. Risk of spoofed server is negligible for a local-only desktop app. |
| T-03-03-02 | Tampering | LLM prompt injection via crafted .docx | mitigate | GBNF grammar constrains LLM output to valid JSON structure only (no prose, no code). jsonDecode + schema validation rejects structurally valid but semantically invalid fields. Post-parse canonicalization strips non-letter characters from answer. |
| T-03-03-03 | Denial of Service | llama.cpp port conflict | mitigate | Make serverUrl configurable via provider override. Validate response contains expected fields before constructing ParseCandidate. Connection refused maps to LlmConnectionException with clear user message. |
| T-03-03-04 | Information Disclosure | LLM output in error messages | mitigate | LlmJsonParseException stores rawResponse — this is only written to local parse_log table (local SQLite, not network-accessible). No user PII in question text. |
| T-03-03-05 | Elevation of Privilege | No authentication on localhost | accept | llama.cpp runs unauthenticated on localhost by design. OS firewall restricts access to local machine. No privilege escalation vector — the server runs with the same user permissions as the app. |
</threat_model>
