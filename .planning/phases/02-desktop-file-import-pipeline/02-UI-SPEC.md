# Phase 02 — UI Design Contract

> Visual and interaction contract for the Desktop File Import Pipeline phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Extends Phase 1's Material 3 foundation with import flow screens.

---

## Phase Scope (UI-Relevant)

This phase delivers the complete file import pipeline UI: platform-branched entry screen, parse progress, question preview/editor, and import summary. It also adds the FAB to HomeScreen and integrates drag-and-drop on desktop.

**Owned requirements** (per REQUIREMENTS.md traceability):
- **IMP-01** — Desktop user selects `.docx` file → imported as question bank
- **IMP-02** — Desktop user selects `.pdf` file → imported as question bank
- **IMP-04** — Parse progress shown with failure reasons; retry on failure
- **QST-03** — Single-choice vs multiple-choice correctly labeled in bank list and display
- **UI-04** — Import page branches by platform: desktop shows `.docx`/`.pdf`/`.json`; Android shows only `.json`

**What this phase also delivers:**
- `.doc` (Word 97-2003) import — scope expansion from CONTEXT.md D-05/D-06
- Drag-and-drop file import on desktop (CONTEXT.md D-03)
- FAB on HomeScreen as primary import trigger (CONTEXT.md D-01)

**What this phase does NOT include:**
- LLM parsing (Phase 3)
- JSON import/export implementation (Phase 5)
- Settings screen (Phase 6)
- Diagnostic pack UI (Phase 6)

**UI surface delivered in Phase 2:**
1. **HomeScreen** — FAB addition (one icon button, bottom-right)
2. **ImportScreen** (rewrite) — platform-branched format picker
3. **ImportProgressScreen** (new) — progress bar + cancel
4. **ImportPreviewScreen** (new) — question list + edit/delete
5. **ImportSummaryScreen** (new) — success/skip counts + retry
6. Drag-and-drop overlay (desktop only)

## Typography

Phase 1 roles carry forward. Phase 2 additions:

| Role | Size | Weight | Line Height | Usage (Phase 2) |
|------|------|--------|-------------|------------------|
| Display | 36px | w400 | 1.2 | Unused in Phase 2 (reserved) |
| Heading | 24px | w500 | 1.25 | Import screen "导入题库" AppBar; Summary stats numbers |
| Body | 16px | w400 | 1.5 | Format tile subtitles; question stem text in preview |
| Label | 14px | w500 | 1.43 | Format tile titles; progress percentage; question index; option keys |
| Caption | 12px | w400 | 1.33 | **New for Phase 2**: parse warning text; "文件大小: 120 KB" metadata; error details |

**Why Caption (12px) is added:**
Phase 2 introduces metadata-heavy screens (file info, parse stats, per-question warnings). 12px caption text is Material 3's `bodySmall` and is reserved exclusively for non-actionable metadata. It does NOT replace Label (14/500) for interactive elements. This addition is justified under the 4-size budget expansion rule ("add a new size only when the phase introduces a new information density tier").

## Pitfall Mitigation (UI-Relevant — Phase 2)

| Pitfall | Phase 2 UI Mitigation |
|---------|----------------------|
| **PITFALL 6** (stem fragmentation / question chunking errors) | ImportPreviewScreen shows every parsed question with edit affordance; user corrects wrong splits before DB commit. Raw text preserved for re-parsing. |
| **PITFALL 1** (LLM JSON drift — not Phase 2, but heuristic parser is the deterministic baseline) | Heuristic parser has no drift risk. Preview screen lets user manually fix wrong answers. |
| **UX anti-pattern: import with no summary** | ImportSummaryScreen shows exact counts: "✅ 成功 98, ⚠ 跳过 2" with per-failure retry. |
| **UX anti-pattern: blocking UI during parse** | Parse runs in background isolate; ImportProgressScreen remains responsive (cancel button, back navigation). Cancel confirmation dialog prevents accidental data loss. |
| **UX anti-pattern: "导入成功" but questions wrong** | Preview screen is mandatory — user MUST review before DB commit. "提交到题库" is the explicit gate. |
