## Spacing Scale

Declared values (8-point scale, from existing codebase conventions):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Compact text spacing (subtitle gap), tight inline gaps |
| sm | 8px | Within-card content gaps, chip-to-text spacing |
| md-sm | 12px | Card list gap (between cards in vertical list) |
| md | 16px | Default element spacing, card inner padding, section body gap |
| lg | 24px | Section break, vertical scroll padding, app body top/bottom |
| xl | 32px | Horizontal margin on expanded layouts (>=840px) |

Exceptions: None. Inline `EdgeInsets` / `SizedBox` using these values only.

**Note for planner/executor:** The codebase uses inline `const EdgeInsets.all(16)` and `const SizedBox(height: 12)` rather than named spacing constants. This contract enforces the values above but does not require extracting them into named tokens — inline is the established project convention.

## Typography

All font sizes and weights come from Material 3 `textTheme` with Noto Sans SC. The following are the roles used in existing code + new Phase 5 screens:

| Role | M3 Role | Effective Size | Weight | Line Height | Usage |
|------|---------|---------------|--------|-------------|-------|
| Display | — | — | — | — | Not used in v1 |
| Heading | `headlineSmall` | ~24px | 400 (regular) | M3 default (~1.2) | Section headers (题库, 复习模式, 数据统计, 题库详情) |
| Title | `titleMedium` | ~16px | 500 (medium) | M3 default (~1.5) | Card titles (题库名, 模式名, 统计卡片标题) |
| Body | `bodyLarge` | ~16px | 400 (regular) | M3 default (~1.5) | Quiz question stems (with optional `w700` for emphasis) |
| Body-Sub | `bodyMedium` | ~14px | 400 (regular) | M3 default (~1.5) | Card subtitles, secondary descriptions |
| Label | `bodySmall` | ~12px | 400 (regular) | M3 default (~1.4) | Drag-drop hints, secondary labels, metadata |

Font weights used:
- **Regular (400):** Default for body, headings, subtitles, labels
- **Semibold (600/w600):** Drop-target hint text ("释放以导入")
- **Bold (700/w700):** Quiz stem emphasis, option letter keys, badge counts

**Rule:** Never go above 700 weight. `900` (black) is banned.

## Color

Material 3 ColorScheme from `ColorScheme.fromSeed(seedColor: Color(0xFF6750A4))` — a purple-toned palette. On platforms where `dynamic_color` is available, this is harmonized with the OS accent color.

### Semantic Color Contract (M3 Roles)

| Role | M3 Token | Usage |
|------|----------|-------|
| Dominant (60%) | `scheme.surface` | Scaffold background, app bar background |
| Secondary (30%) | `scheme.surfaceContainerHighest` | Card backgrounds, option cards (normal state), letter backgrounds |
| Accent (10%) | `scheme.primary` | FilledButton background, selected option state (primaryContainer), option letter foreground when selected, drag-over border, model card borders |
| Accent (on-primary) | `scheme.onPrimary` | Button text |
| On-surface | `scheme.onSurface` | All body text, card text, icons |

### Fixed Semantic Colors (Not From ColorScheme)

| Color | Usage | Reserved For |
|-------|-------|-------------|
| `Color(0xFFD32F2F)` (Red) | Wrong-answer border (2px), cancel icon, wrong question chip badge background | Incorrect answer feedback ONLY |
| `Colors.green` / `Colors.green.shade500` | Correct-answer background tint (0.22/0.14 opacity), check_circle icon | Correct answer feedback ONLY |
| `scheme.error` / `scheme.onError` | Destructive action hint, badge counter color | Error states and wrong-count badges |

### Accent Reserved For

Accent (`scheme.primary` / `scheme.primaryContainer`) is reserved for:
1. Primary action buttons (`FilledButton`)
2. Selected option state in quiz (`OptionCardState.selected`)
3. Option letter foreground in selected state
4. Drag-drop visual feedback overlay (border + text)
5. Interactive element hover/focus indicators

**Never use accent** for: decorative backgrounds, non-interactive text, section dividers, or icon-only decorations that are not interactive.

### 3. StatsScreen (Full Implementation)

**Current state:** Placeholder with `Text('TODO — StatsScreen...')`.

**AppBar:** `AppBar(title: "数据统计")`.

**Body layout:** LayoutBuilder + ConstrainedBox(720) + SingleChildScrollView pattern:

**States:**

| State | What User Sees |
|-------|---------------|
| Loading | Centered `CircularProgressIndicator` + "加载统计..." text |
| Empty (no banks / no attempts) | Centered Column: `Icon(insights_outlined, size: 48, color: onSurface.withOpacity(0.4))` + SizedBox(16) + Text("暂无统计数据", headlineSmall) + SizedBox(8) + Text("完成答题后这里会显示各题库的正确率统计", bodyMedium, color: onSurface.withOpacity(0.6)) |
| Error | Centered Column: `Icon(error_outline, size: 48, color: error)` + SizedBox(16) + Text("加载失败", headlineSmall) + SizedBox(8) + Text("请返回重试", bodyMedium) + SizedBox(24) + OutlinedButton("重试") |
| Data | Vertical list of `_StatsBankCard` widgets (12px gap) |

**Stats bank card (`_StatsBankCard`) — D-09, D-10:**
```
Card (elevation: 1, borderRadius: 12)
  InkWell (onTap → toggle expand, borderRadius: 12)
    Padding(all: 16)
      Column
        ├─ Row
        │    ├─ Expanded
        │    │    Column(crossAxisAlignment: start)
        │    │      ├─ Text(bankName, headlineSmall)
        │    │      └─ Text("总题数 N · 总答题次数 M", bodyMedium, muted)
        │    └─ AnimatedRotation(icon: Icons.expand_more, turns: expanded ? 0.5 : 0)
        ├─ SizedBox(height: 12)
        ├─ Row(mainAxisAlignment: spaceAround)
        │    ├─ _StatChip("正确率", "78%")   ← primary color emphasis
        │    └─ _StatChip("错题本", "12")     ← error color if >0, muted if 0
        ├─ if (expanded)
        │    SizedBox(height: 12)
        │    Divider(height: 1)
        │    SizedBox(height: 8)
        │    Column (12px gap)
        │      ├─ _PerModeRow("乱序抽题", attemptCount, correctRate)    ← e.g. "32次 · 78%"
        │      ├─ _PerModeRow("错题复习", attemptCount, correctRate)    ← e.g. "15次 · 92%"
        │      └─ _PerModeRow("错题抽查", attemptCount, correctRate)    ← e.g. "8次 · 63%"
```

**D-11: No charts.** Pure text + numbers. No bar charts, no pie charts, no sparklines.

**Per-mode breakdown (D-10):** Each `_PerModeRow` shows:
- Mode name (乱序抽题 / 错题复习 / 错题抽查)
- Attempt count ("N次")
- Correct rate percentage ("X%")

When a mode has 0 attempts, show "暂无" instead of "0%".

### 4. QuizScreen — Multiple-Choice Exact Match

**Change from current:** The quiz screen already supports multi-choice via checkbox toggle + confirm button (`isMultiChoice = correctKeys.length > 1`). Phase 5 adds exact-match grading logic.

**UI unchanged:** No visual changes to QuizScreen. The existing checkbox toggle + confirm button flow is correct.

**Backend change (D-07, D-08):** The `submitAnswer()` grading logic must enforce exact-match:
- Single-choice (`correctKeys.length == 1`): `givenKeys.toSet() == correctKeys.toSet()` (already correct)
- Multi-choice (`correctKeys.length > 1`): `givenKeys.toSet() == correctKeys.toSet()` — all correct AND no extras

**No partial credit.** User must select exactly the correct answer set. Missing or extra options = incorrect.

### 5. ImportScreen — JSON Import Integration (D-05, D-06)

**Current state:** Desktop import screen already has a "JSON 题库" tile (`FileFormatTile`) with file picker. The tile code at line 210-214 invokes `_pickJsonFile()` which calls `_onFileSelected()`.

**Change (D-05):** When `_startParseAndNavigate()` processes a `.json` file, the pipeline must:
1. Read and validate JSON against the established format (numbered objects, `answer` option map, `key` string, `answer_type` 0/1)
2. Parse directly into DB (no editing preview) — skip the import preview/editing phase
3. Navigate directly to import summary (skip `ImportPhase.editing`)

**Change (D-06):** When importing a bank whose name matches an existing bank:
- Replace the existing bank silently (delete old + insert new in a single transaction)
- No confirmation dialog
- Show SnackBar "已替换已有题库「bankName」"

**UI unchanged.** The existing import screen tiles and file picker are sufficient. No new buttons or screens needed.

## Interaction Contract

### Desktop Keyboard

| Context | Key | Action |
|---------|-----|--------|
| Quiz: select option | A / B / C / D | Selects or toggles option (unchanged, existing) |
| Quiz: confirm submit | Space | Confirms answer in confirm mode (unchanged, existing) |

| Bank detail: export | — | Mouse click on button (no keyboard shortcut) |

### Hover States (Desktop Mouse)

- **Card + InkWell:** M3 default hover overlay (built-in, no custom handling needed)
- **FilledButton / FilledButton.tonal:** M3 default hover state
- **Stats expand card:** Cursor changes to pointer on InkWell area
- **No custom hover effects needed** — M3 default behavior is sufficient

### Touch/Click Targets

- All interactive elements use M3 default minimum tap target (48px). Buttons already meet this via `minimumSize: Size(64, 40)` from theme.
- Stats expand toggle: the entire card header row is tappable (not just the chevron icon)
