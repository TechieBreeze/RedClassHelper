<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dart:ffi → llama.cpp native code | FFI boundary — native code runs in same process as Dart VM. Crashes in llama.cpp kill the entire app. |
| Model file → llama.cpp loader | Model loaded from user's filesystem — path is trusted (user controls it) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08-01 | Denial of Service | ffi_llm_client.dart | mitigate | FFI crash kills the Dart VM — unacceptable for production. Mitigation: run inference on a separate isolate (Dart isolates isolate memory but not native code). If isolate crashes, the main app continues. Wrap FFI calls in try/catch where possible. |
| T-03-08-02 | Tampering | llama.cpp shared library | accept | The .dll/.so is user-installed. A compromised shared library could execute arbitrary code. Accept risk — the user controls their system; this is equivalent to running any native application. |
| T-03-08-03 | Elevation of Privilege | DynamicLibrary.open | accept | DynamicLibrary.open loads a native library into the process. The library runs with the same privileges as the app. Accept risk — standard FFI behavior with no additional privilege escalation. |
</threat_model>
