# RedClass (红课复习)

## What This Is

RedClass 是一个闭源的本地化大学课程复习工具，专为"导入老师发放的题库文件 → 自动解析建库 → 多模式刷题复习"这一完整链路而设计。学生将老师下发的 `.docx` / `.pdf` 题库文件导入应用，由应用调用本地轻量 LLM 自动解析为结构化题目数据集；之后用户可在**乱序抽题**、**错题复习**、**错题抽查**三种模式间切换复习，三种模式间错题本互通联动（乱序产生的错题进入错题本，错题复习中答对的题目进入"已掌握"并从抽查范围中移除）。最终产物为可在 Windows 上运行的 `.exe` 与可在 Android 上运行的 `.apk`，同一套 Flutter 代码同时产出双端。

## Core Value

把"老师发的题库文件"零摩擦地变成"可立刻投入复习的结构化题库"，让本地刷题体验比任何在线刷题网站都更顺手——**离线可用、零配置、解析即用**。

### Out of Scope

<!-- 显式排除：含排除理由，防止后续被反复提议 -->

- **云端同步 / 多设备同步** — 个人/小范围分享工具，无后端，单设备本地化最简
- **多用户登录 / 注册** — 无需账户，开箱即用，避免增加复杂度
- **题库中的图片/公式渲染** — 老师题库主要是文字，图片会引入 OCR + 资源管理复杂性，超出 v1
- **章节 / 知识点筛选** — 题库结构不一定有清晰章节标签，强加会引入错配
- **题目图片提取** — 与上一项配套排除
- **填空题 / 判断题 / 简答题** — 解析复杂度与判分逻辑不在 v1 范围
- **在线题库共享社区** — 闭源小工具，无社交/分享模块
- **暗色 / 浅色主题切换** — Material 3 默认主题即可，主题切换属 v2
- **字体大小调节** — 默认字号 + 平台原生缩放支持即可

## Context

- **用户场景**：大学生期末复习，老师通常会以 `.docx` 或 `.pdf` 形式发放题库（含答案）；目前市面上多数刷题 App 都是为考研/公考设计的"题库已内置"模式，没有"导入自有题库"路径
- **痛点**：手动整理 Word/PDF 题库 → 复制粘贴到 Excel/Anki 耗时巨大；通用刷题 App 不能复用老师给的真实题库
- **目标受众**：本项目自用 + 同班同学分享（小范围），无需考虑规模化
- **运行环境**：开发者在 Windows 上开发，目标用户在 Windows + Android 上使用
- **UI 设计参考**：用户的 `ui-ux-pro-max` skill（`C:\Users\Lenovo\.claude\plugins\cache\ui-ux-pro-max-skill\ui-ux-pro-max\2.5.0`），在 UI 设计阶段作为视觉规范来源
- **已知技术决策**：Flutter 是核心，因为单代码库双端交付；本地 LLM 解析是关键，因为题库格式"有规律但非完全固定"，纯正则不可靠

## Constraints

- **Tech stack**：Flutter（Dart）— 必须同时支持 Windows 与 Android 打包
- **本地化**：数据全部存在本地，无后端服务依赖
- **离线**：网络断开后所有功能仍可用（除首次下载 LLM 模型）
- **闭源**：项目本身闭源，源码不公开发布；题库数据归用户所有
- **解析依赖**：使用本地 LLM（小型模型即可，题库规律性强）；模型需可在 Windows 与 Android 端运行
- **隐私**：用户的题库文件、错题、收藏不会离开本机
