## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pub.dev → project | Untrusted package code via `pub get`; could execute arbitrary build scripts in pubspec authors' packages |
| Build script → host machine | `build_runner` runs Dart code in `pubspec.yaml`'s `executables` and tool callbacks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering / EoP | `pubspec.yaml` | mitigate | Pin all versions to `^X.Y.Z` ranges matching the verified STACK.md versions; do NOT use unconstrained `any` or `latest` for any production dep |
| T-01-02 | Repudiation | `flutter create` overwriting `.planning/` | mitigate | `flutter create .` only writes `lib/`, `test/`, platform folders, `pubspec.yaml`, etc. — it does NOT touch `.planning/` (a top-level custom dir). Verified by re-listing `.planning/PROJECT.md` after `flutter create` |
| T-01-03 | Information Disclosure | `build_runner` scripts | accept | `build_runner` is a published pub.dev package (Flutter team) — same trust level as the Dart SDK itself. No additional mitigation needed in Phase 1 |
| T-01-04 | T / EoP | `flutter pub add --dev` writing to pubspec | mitigate | Plan hand-edits `pubspec.yaml` (NOT `flutter pub add`) so versions match STACK.md exactly — avoids accidental upgrades to incompatible versions |
</threat_model>
