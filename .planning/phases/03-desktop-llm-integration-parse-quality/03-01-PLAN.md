<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provider→Android | llmClientProvider gates all LLM access on Platform.isWindows \|\| Platform.isLinux; Android gets UnsupportedError |
| provider→LlmClient | llmClientProvider resolves to a concrete LlmClient via switch on LlmMode — no untrusted input at this layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | llmClientProvider | mitigate | @Riverpod(keepAlive: true) prevents provider from being garbage-collected mid-parse; switch expression is exhaustive so no fallthrough to wrong implementation |
| T-03-02 | Tampering | LlmMode enum | accept | LlmMode is a simple enum with no serialization/deserialization; no user-controlled path to modify it outside the provider override system |
| T-03-03 | Information Disclosure | LlmError messages | mitigate | Error messages include serverUrl — this is localhost:8080 in production, so no sensitive internal network info is exposed. Exception.toString() is only logged locally to parse_log table. |
</threat_model>
