<task name="task-heuristic-parser-isolate">
<read_first>
- file:.planning/phases/02-desktop-file-import-pipeline/02-RESEARCH.md §6 — Heuristic parser design

- file:.planning/phases/02-desktop-file-import-pipeline/02-RESEARCH.md §8 — Edge cases
</read_first>
<action>
1. Create `lib/features/import/parse/` directory.
2. Create `lib/features/import/parse/question_model.dart`:
   - `ParsedQuestion` data class (freezed):
     - `int index` — original position in file
     - `String stem` — question body
     - `List<String> options` — A/B/C/D (or A/B/C/D/E, etc.)
     - `Set<String> correctAnswers` — detected answers (e.g. {"A", "B"})
     - `QuestionType detectedType` — single / multiple (enum)
     - `List<String> warnings` — parse warnings (e.g. "答案未识别")
3. Create `lib/features/import/parse/heuristic_parser.dart`:
   - `Future<List<ParsedQuestion>> parseQuestions(String rawText)`
   - **Step 1 (Normalize):** Strip BOM, normalize newlines to `\n`, collapse >2 consecutive newlines
   - **Step 2 (Split into question blocks):** Regex: `/(?:^|\n)(?:\d+[\.\)、]|[一二三四五六七八九十]+[、\.]|[（(]\d+[)）])/gm` — detect question number patterns
   - **Step 3 (Extract stem):** Everything from question number to first option marker. Option marker regex: `/[A-Z][\.\)、]/` (standing after a newline)
   - **Step 4 (Inline answer detection):** Search stem for patterns like `（D）` `(ABC)` `（A、B、C）` — Chinese/Spanish/ASCII parentheses. Extract letters, remove from stem.
   - **Step 5 (Separate answer block detection):** If no inline answer found, search for standalone "答案" / "参考答案" / "【答案】" blocks after last option
   - **Step 6 (Extract options):** Match `/[A-Z][\.\)、]\s*(.+?)(?=\n[A-Z][\.\)、]|\n\n|$)/s` for each option per question
   - **Step 7 (Type detection):** If `correctAnswers.length > 1` OR stem contains "多选"/"多选题" OR option count > 4 AND no single-choice keywords → `QuestionType.multiple`. Else `QuestionType.single`.
   - **Step 8 (Warnings):** Flag: no answer detected, stem < 5 chars, only 2 options, duplicate option text, answer letter not in options
   - **Step 9 (Filter empties):** Drop any block where stem is empty AND options are empty
   - Return list with original file order preserved
4. Create `lib/features/import/parse/parse_isolate.dart`:
   - Top-level entry: `Future<List<Map<String, dynamic>>> parseIsolateEntry(Map<String, dynamic> args)`
   - Accepts: `{"rawText": "...", "jobId": "..."}`
   - Calls `HeuristicParser.parseQuestions()`, returns JSON-serializable list
   - Progress reporting: sends `{"type": "progress", "current": N, "total": M, "stage": "splitting"|"extracting"|"done"}` via `SendPort`
   - Error reporting: sends `{"type": "error", "message": "..."}` on exception
5. Create `lib/features/import/parse/parse_job.dart`:
   - `ParseJob` class (not freezed — mutable state machine):
     - `String jobId` (uuid v4)
     - `String filePath`
     - `String fileExtension`
     - `ParseStatus status` (pending | extracting | parsing | done | cancelled | error)
     - `List<ParsedQuestion>? questions`
     - `String? errorMessage`
     - `double progress` (0.0–1.0)
     - `StreamController<ParseStatusUpdate>` for progress events
6. Write comprehensive parser tests:
   - Standard numbered format: `1. ... A. ... B. ... C. ... D. ...`
   - Chinese-numbered: `一、 ... A、 ... B、 ...`
   - Inline answer: `...核心是（A）的政治立场。`
   - Answer-block style: `...\n【答案】A\n`
   - Multi-choice: `...（ABC）` / `...多选题...`
   - Edge cases: empty file, no questions found, 2-option questions, 5+ options, duplicate stem
   - Chinese university political question format (from sample files)
   - All sample files in `doc/example/` — verify each produces ≥1 valid ParsedQuestion
</action>
